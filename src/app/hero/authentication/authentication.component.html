Angular Authonication uing Auth-Guard(Route Guards)

Description

  • Angular Authonicate user by using Auth-Guard which in tern immplements Route-Guard
  • Angular Router are interface , this router - gurad tells whether or not should allow the navigation to requested route.
  • router gurd works based on ture or false return form the authService , if its ture then its allow navigation to requested path else restrict the navigation.
  • In router-guard we have 5 types of authonication as
    1. canActive : gives navigate permison for user only for components if authService return ture
    2. canActiveChild : prvents the user to navigate any path of its childComponent
    3. canDeactive : to prevent the user to navigate
    4. canLoad : prevents the application form loading entire module if the user is not authorized
    5. resolve : loads the necessury data form the route ids i.e route parameter , asynchronously loads the data and have ready it by component get initilize
  • we can restrict the for perticular page or entire panel based on route gurd.
  • create auth.service and auth guard files under service using
    ng g s services/auth/auth
    ng g s services/auth/auth
  • after login , save token in local stroage using localStroage.setItem()
  • in auth.service.ts file check the token is set or not , if token is avaliable then return ture ,
  • in auth-guard.service.ts , in canActivate method call this checkToken method of authService , if it ture return ture else naviagte back to login page
  • in app-routing.modules.ts file , here add canActivate in path where you want prevent the access .
