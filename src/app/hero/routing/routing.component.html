Routing

To handel the navigation from one view to another we use the angular routing

Description

  • application get loads with index.html
    which has {{base}} helps to components how to navigate with url ,
    and {{app}} which is the main root link for navigation
  • app.module.ts file we import the appRouting module . and all the components list in declartion tag.
  • app-routing.module.ts which has router=[] here we declare all the routing paths
    • {{routerDirective}} : available form router pacakge and routing goes form here .
    • routerLink : takes the link which is defined in router array and based on url its navigates
    • routerLinkActive : which hold the active class and we can define styles for it
    • path : path of component
  • Child Routing
    Some routes may view in only other routes that cases we use child routing.
    Example : after navigationg to listEmploee we need over view and concat info this should to navigate after the navigation to listEmployee.
  • Lazy Loading
    the very first time server get loads along with all the componnent we listed in app.module.ts get loads this includes a lot of data loads from server . to avoid this load the only need modules and then navigate . this concept using in admin panels
